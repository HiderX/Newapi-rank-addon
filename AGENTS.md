# 项目协作规范

## 交流与验证

- 默认使用简体中文交流。
- 声称完成、修复、通过前，必须先运行真实验证命令并核对输出。
- 代码改动应优先使用现有架构和测试模式，避免无关重构。

## 固定发布流程

完成代码或文档改动后，除非用户明确要求暂停、只改不提、只本地验证或不部署，默认自动执行：

1. README：检查 README 是否需要同步更新；涉及配置、使用方式、部署流程、主题结构或用户可见行为时必须更新。
2. 验证：运行与改动相关的测试；涉及发布时运行全量测试。
3. 提交：把本次相关改动提交到当前开发分支。
4. Review：提交后执行代码审查，处理 Critical/Important 问题后再继续。
5. Merge：审查通过后合并到 `main`。
6. 部署：`main` 更新后部署到生产服务器 `oracle01`，并确认服务状态。

上述流程是本项目的默认收尾流程，后续完成改动时不需要用户再次提醒。
